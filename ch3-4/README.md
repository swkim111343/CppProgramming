## C++ 챕터 3-4 실습과제 1
1. 접근지정자가 왜 필요한가?
    - 데이터와 함수에 대한 접근 범위를 제한하기 위해 필요
    - 외부에서 함부로 데이터 변경 방지
    - 객체의 안정성 유지
    - 잘못된 사용(오용) 방지

 2. 생성자와 setter 멤버함수는 모두 멤버변수를 설정하는 기능이다. 차이점을 설명하라.

| 구분    | 생성자 (Constructor) | setter           |
| ----- | ----------------- | ---------------- |
| 실행 시점 | 객체 생성 시 자동 실행     | 객체 생성 후 필요할 때 호출 |
| 목적    | 초기값 설정            | 값 변경             |
| 호출 방식 | 자동                | 직접 호출            |
| 호출 횟수 | 객체당 1번            | 여러 번 가능          |
| 반환형   | 없음                | 보통 void          |
| 특징    | 객체 생성과 동시에 실행     | 특정 값만 선택적으로 변경   |


3. 객체지향형 언어의 특징인 캡슐화와 정보은닉을 설명하라.
    - 캡슐화 (Encapsulation) : 데이터 + 기능(함수)을 하나로 묶는 것
    - 정보은닉 (Information Hiding) : 외부에서 내부 데이터를 직접 못 보게 숨기는 것

- 캡슐화(데이터와 함수가 하나로 묶임)
<pre>class Car {
private:
    int speed;

public:
    void accelerate() {
        speed += 10;
    }
};</pre>

- 정보은닉(외부에서 speed 직접 수정 불가)
<pre>private:
    int speed;</pre>
