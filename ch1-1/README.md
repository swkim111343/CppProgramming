## 실습과제 1

1. 강의노트에서 몰랐던 용어에 대하여 자세히 조사하시오.
    - 어셈블리어 : 기계어의 명령을 ADD, SUB, MOVE 등과 같은 상징적인 니모닉 기호(mnemonic symbol)로 일대일 대응시킨 언어
    - 어셈블러 : 어셈블리어 프로그램을 기계어 코드로 변환하는 프로그램
    - 목적 파일(object file) : 기계어로 변환된 파일, 확장자는 .obj (예: hello.obj)
    - 실행 파일(executable file) : 실행이 가능한 기계어로 작성된 파일, 확장자가 .exe (예: hello.exe)
    - 링커(linker) : 여러 개의 목적 파일과 라이브러리를 결합하여 하나의 실행 파일을 생성하는 프로그램

2. 알고리즘 설계와 코딩 중 무엇이 중요하고 그 이유는 무엇인가?
    - 코딩 능력도 중요하지만 알고리즘 설계가 더 중요하다.
        → 이유는 코딩을 아무리 잘해도 알고리즘 설계가 비효율적이면 실행 속도가 느려지고 자원 낭비가 발생하기 때문이다.

3. 클래스와 객체가 무엇인지 설명하시오.
    - 클래스란 객체를 만들기 위한 설계도(틀)이다.
    - 객체란 클래스를 기반으로 실제 생성된 실체(인스턴스)이다.

4. 절차지향 프로그래밍 언어와 객체지향 프로그래밍 언어의 차이를 조사하라.
    - 절차지향은 함수 중심으로 프로그램을 구성한다.
    - 객체지향은 객체 중심으로 프로그램을 구성한다.
    - 유지보수 측면에서 절차지향은 프로그램이 커질수록 복잡해지기 쉽고,
      객체지향은 재사용성과 확장성이 높아 유지보수가 용이하다.

5. 라이브러리가 무엇인지 자세히 설명하시오.
    - 개발자가 모든 기능을 직접 구현하지 않고, 필요한 기능을 가져다 사용할 수 있도록 만든 재사용 가능한 코드들의 묶음이다.

6. 라이브러리가 목적 파일 형태로 존재하는 이유를 설명하라.
    - 컴파일 시간을 줄이기 위해서이다.
    - 이미 컴파일된 형태로 제공함으로써 재사용이 가능하고 배포가 편리하다.

7. 링크 과정이 필요한 이유를 설명하라.
    - 여러 개의 목적 파일과 라이브러리를 하나로 결합하여 실행 파일을 만들기 위해 필요하다.
